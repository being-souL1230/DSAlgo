Introduction to Graph Traversal

What are DFS and BFS?

Depth-First Search (DFS) and Breadth-First Search (BFS) are fundamental graph traversal algorithms used to systematically visit all nodes in a graph or search for a specific node. These algorithms form the backbone of many important applications in computer science, from pathfinding to web crawling.

Core Concepts

  • Graph Traversal: Systematically visiting each node in a graph exactly once
  • Search Algorithms: Finding a path from start node to goal node
  • Data Structures: Queue (FIFO) for BFS, Stack (LIFO) for DFS
  • Exploration Strategy: DFS goes deep first, BFS explores level by level

Tree vs Graph Context

While both algorithms work on general graphs, they behave differently on trees (acyclic connected graphs):

On Trees (No Cycles):
  • BFS: Level-order traversal
  • DFS: Pre/in/post-order traversal
  • Both find unique paths
On General Graphs:
  • BFS: Shortest path in unweighted graphs
  • DFS: May find longer paths
  • Cycle detection needed

Breadth-First Search (BFS)

How BFS Works

How BFS Works (Step by Step)

  1. Initialize: Create a queue and visited set, enqueue start node
  2. Process Level: Dequeue front node, mark as visited, process it
  3. Explore Neighbors: Enqueue all unvisited neighbors of current node
  4. Repeat: Continue until queue is empty or goal is found
Example Tree:
     A
   /   \
  B     C
 / \   / \
D   E F   G
BFS Traversal Order: A → B → C → D → E → F → G

BFS Implementation Variants

Recursive BFS:
  • Less common
  • Uses system call stack
  • Harder to implement
  • Not recommended
Iterative BFS:
  • Most common approach
  • Uses explicit queue
  • Easier to understand
  • Better performance

Real-World Analogy

Imagine you're at a party looking for someone. BFS is like asking everyone in the room if they're the person, then moving to the next room. You check all people at the same "distance" from you before going further. It's like ripples spreading in a pond - the wave expands outward evenly in all directions.

Complexity Analysis

Time Complexity: O(V + E)
where V = vertices, E = edges. We visit each vertex once and explore each edge once.

Space Complexity: O(V)
Queue can hold up to V vertices in the worst case (all nodes at one level).

✅ Best For: Finding shortest path, level-order traversal, finding all nodes within a given distance

Depth-First Search (DFS)

How DFS Works

How DFS Works (Step by Step)

  1. Initialize: Create a stack and visited set, push start node
  2. Process Node: Pop top node from stack, mark as visited, process it
  3. Explore Deep: Push all unvisited neighbors onto stack (in reverse order)
  4. Backtrack: Continue until stack is empty or goal is found
Example Tree:
     A
   /   \
  B     C
 / \   / \
D   E F   G
DFS Traversal Order: A → B → D → E → C → F → G

DFS Implementation Variants

Recursive DFS:
  • Most natural implementation
  • Uses system call stack
  • Easier to understand
  • Risk of stack overflow
Iterative DFS:
  • Uses explicit stack
  • Avoids stack overflow
  • Slightly more complex
  • Better for large graphs
Tree Traversal Types (DFS variants):
  • Pre-order: Visit root, then left subtree, then right subtree
  • In-order: Visit left subtree, root, then right subtree
  • Post-order: Visit left subtree, right subtree, then root

Real-World Analogy

Imagine exploring a maze or a cave system. DFS is like following one path as far as it goes, marking your way. When you hit a dead end, you backtrack to the last junction and try another path. It's like exploring a rabbit hole - you go deep into one topic before moving to another. Or think of file system search - diving deep into folders before checking siblings.

Complexity Analysis

Time Complexity: O(V + E)
where V = vertices, E = edges. Each vertex and edge is explored once.

Space Complexity: O(V)
Recursive implementation: O(h) call stack where h is max depth (can be V in worst case).
Iterative implementation: O(V) for explicit stack to track nodes to visit.

✅ Best For: Detecting cycles, topological sorting, finding connected components, maze solving

BFS vs DFS: Comparison & Applications

BFS vs DFS: Comprehensive Comparison

Algorithm Characteristics

Breadth-First Search (BFS)
  • Data Structure: Queue (FIFO)
  • Traversal: Level by level
  • Path Quality: Shortest path guaranteed
  • Memory: High (stores all current level)
  • Implementation: Iterative preferred
  • Use Case: Unweighted shortest path
Depth-First Search (DFS)
  • Data Structure: Stack (LIFO)
  • Traversal: Deep exploration first
  • Path Quality: May find longer paths
  • Memory: Low (stores current path)
  • Implementation: Recursive common
  • Use Case: Topological sort, cycle detection

When to Choose Which Algorithm

Choose BFS when:
  • You need the shortest path
  • Graph is unweighted
  • Answers are closer to root
  • Memory is not a constraint
  • Level-order processing needed
Choose DFS when:
  • Memory is limited
  • Any path will do
  • Need topological ordering
  • Detecting cycles
  • Solving puzzles/games

Important Considerations

  • Graph Size: DFS works better on large graphs due to lower memory usage
  • Graph Density: BFS performs better on sparse graphs (fewer edges)
  • Goal Location: BFS is better when goal is likely close to start
  • Path Existence: DFS may find a path faster in some cases
  • Cycle Handling: Both need visited tracking to prevent infinite loops

Code Implementation

BFS Implementation

def bfs(graph, start, goal):
    visited = set()
    queue = [[start]]
    while queue:
        path = queue.pop(0)
        node = path[-1]
        if node == goal:
            return path
        if node not in visited:
            visited.add(node)
            for neighbor in graph.get(node, []):
                new_path = list(path)
                new_path.append(neighbor)
                queue.append(new_path)
    return None

BFS Implementation

public List<Integer> bfs(Map<Integer, List<Integer>> graph, int start, int goal) {
    Set<Integer> visited = new HashSet<>();
    Queue<List<Integer>> queue = new LinkedList<>();
    queue.offer(Arrays.asList(start));
    while (!queue.isEmpty()) {
        List<Integer> path = queue.poll();
        int node = path.get(path.size() - 1);
        if (node == goal) return path;
        if (!visited.contains(node)) {
            visited.add(node);
            for (int neighbor : graph.getOrDefault(node, new ArrayList<>())) {
                List<Integer> newPath = new ArrayList<>(path);
                newPath.add(neighbor);
                queue.offer(newPath);
            }
        }
    }
    return null;
}

BFS Implementation

vector<int> bfs(unordered_map<int, vector<int>>& graph, int start, int goal) {
    unordered_set<int> visited;
    queue<vector<int>> q;
    q.push({start});
    while (!q.empty()) {
        vector<int> path = q.front();
        q.pop();
        int node = path.back();
        if (node == goal) return path;
        if (visited.find(node) == visited.end()) {
            visited.insert(node);
            for (int neighbor : graph[node]) {
                vector<int> newPath = path;
                newPath.push_back(neighbor);
                q.push(newPath);
            }
        }
    }
    return {};
}

BFS Implementation

int* bfs(int graph[][MAX], int n, int start, int goal, int* pathLen) {
    int visited[MAX] = {0};
    int queue[MAX], front = 0, rear = 0;
    int parent[MAX];
    for (int i = 0; i < MAX; i++) parent[i] = -1;
    queue[rear++] = start;
    visited[start] = 1;
    while (front < rear) {
        int node = queue[front++];
        if (node == goal) {
            int* path = malloc(MAX * sizeof(int));
            int idx = 0, curr = goal;
            while (curr != -1) {
                path[idx++] = curr;
                curr = parent[curr];
            }
            *pathLen = idx;
            return path;
        }
        for (int i = 0; i < n; i++) {
            if (graph[node][i] && !visited[i]) {
                visited[i] = 1;
                parent[i] = node;
                queue[rear++] = i;
            }
        }
    }
    return NULL;
}
